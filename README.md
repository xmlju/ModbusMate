# ModbusMate 产品使用说明书

> **版本** 0.1.0 · 通用 Modbus-TCP 调试工具 · Windows / Mac 桌面应用

ModbusMate 是一款面向现场工程师和调试人员的 Modbus-TCP 通信调试工具，支持轮询监控、单点写入、多类型数据解析、线性公式换算和仪表盘视图，帮助快速完成设备调试与数据验证。

## 一、快速上手

### 1.1 下载与安装

**Mac 用户**：双击 `ModbusMate-0.1.0-arm64.dmg`，将应用拖入 Applications 文件夹即可。

**Windows 用户**：运行 `ModbusMate Setup 0.1.0.exe`（NSIS 安装包），或直接双击 `ModbusMate 0.1.0.exe`（便携版，免安装）。

### 1.2 连接设备并开始监控

```
① 输入设备 IP → ② 点击"连接" → ③ 选择区域/起始地址/数量 → ④ 点击"开始监控"
```

连接状态灯变绿表示成功，日志栏会显示"已连接"信息；表格将按设定周期自动刷新数据。

### 1.3 最小系统要求

- **Mac**：macOS 10.12+（ARM64 / x64）
- **Windows**：Windows 7+
- **网络**：与 Modbus-TCP 设备之间二层/三层可达（同网段或路由可达）

## 三、连接设备

### 3.1 连接参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| **IP** | 设备 IP 地址，如 `192.168.1.10` | 空 |
| **端口** | Modbus-TCP 端口 | `502` |
| **从站 ID** | 从站编号（Unit ID） | `1` |

### 3.2 连接与断开

- 点 **「连接」** → 状态灯变 **绿色**，按钮文字变为 **「断开」**
- 再点 **「断开」** → 停止监控并断开 TCP 连接
- 断开后连接参数会自动保存，下次启动自动恢复

### 3.3 错误提示

连接失败时，日志栏和状态灯会给出中文原因：

| 现象 | 含义 |
|------|------|
| `192.168.1.10:502 拒绝连接` | 设备未开启 Modbus-TCP 服务或端口错误 |
| `无响应，请检查设备电源、网线和 IP 设置` | 网络不通或设备未开机 |
| `找不到主机` | IP 地址错误 |

## 四、监控数据

### 4.1 监控配置

连接成功后，在监控配置栏设置：

| 参数 | 说明 | 取值 |
|------|------|------|
| **区域** | 要读取的区域类型 | 保持寄存器 / 输入寄存器 / 线圈 / 离散输入 |
| **起始地址** | 协议地址（从 0 起），勾选 PLC 地址时显示为 4xxxx 格式 | 0~65535 |
| **数量** | 一次读取的寄存器/线圈数 | 1~120 |
| **周期** | 轮询刷新间隔 | 100ms / 500ms / 1s / 2s / 5s / 10s |

### 4.2 四种区域类型

| 区域 | 功能码 | 读写属性 | 写入方式 |
|------|--------|----------|----------|
| 保持寄存器 | FC03 读 / FC06/FC16 写 | 可读可写 | 单字用 FC06，双字用 FC16 |
| 输入寄存器 | FC04 读 | 只读 | 不支持写入 |
| 线圈 | FC01 读 / FC05 写 | 可读可写 | 写 0/1 |
| 离散输入 | FC02 读 | 只读 | 不支持写入 |

### 4.3 变化高亮

数值发生变化的行会以**黄色背景闪烁**约 600ms 提示，方便快速定位变化点。

### 4.4 32 位类型占用

当某行数据类型设为 Int32 / UInt32 / Float32（占 2 个寄存器）时，下一行自动显示"（由上行 32 位占用）"并半透明显示，不参与独立解析。

## 五、数据类型与解析

### 5.1 支持的数据类型

每行可独立设置数据类型和字序：

| 类型 | 标签 | 占用寄存器数 | 说明 |
|------|------|-------------|------|
| **UInt16** | 无符号 16 位整数 | 1 | 原样显示，范围 0~65535 |
| **Int16** | 有符号 16 位整数 | 1 | 补码转换，范围 -32768~32767 |
| **Int32** | 有符号 32 位整数 | 2 | 两个寄存器组合，可选择 AB/BA 字序 |
| **UInt32** | 无符号 32 位整数 | 2 | 两个寄存器组合，可选 AB/BA 字序 |
| **Float32** | 32 位浮点数 | 2 | 两个寄存器按 IEEE754 组合 |
| **Hex** | 十六进制 | 1 | 显示为 `0x00FA` 格式 |

### 5.2 字序（Word Order）

32 位类型占用两个 16 位寄存器（高字 + 低字），不同设备厂家字序不同：

| 字序 | 说明 | 示例（25.6 = 0x41CCCCCD） |
|------|------|-------------------------|
| **AB** | 高字在前（大端序） | 寄存器 N = 0x41CC，寄存器 N+1 = 0xCCCD |
| **BA** | 低字在前（小端序） | 寄存器 N = 0xCCCD，寄存器 N+1 = 0x41CC |

默认为 AB 字序。如果解析值不正常（如数量级明显不对），尝试切换 BA 字序。

### 5.3 原始值列

原始值列始终显示 16 位十六进制格式（如 `0x00FA`），不随数据类型切换而变化。

### 5.4 解析值列

解析值列显示按所选数据类型解码后的值：
- 设置了公式的行显示 **公式转换后的工程值+单位**，鼠标悬停显示原始解析值
- 未设置公式的行直接显示解码值

## 六、写入操作

### 6.1 写入入口

可写区域（保持寄存器 / 线圈）每行右侧有 **「写入」** 按钮；只读区域（输入寄存器 / 离散输入）无此按钮。

### 6.2 写入流程

1. 点击某行的 **「写入」** 按钮 → 弹出写入弹窗
2. 输入数值 → 系统自动按该行数据类型编码
3. 点击 **「写入」** → 下发 → 日志栏显示"写入成功"
4. 下个轮询周期自动回读确认新值

### 6.3 不同类型写入说明

| 类型 | 输入示例 | 说明 |
|------|----------|------|
| 线圈 | `1` 或 `0` | 1=ON，0=OFF |
| UInt16 / Int16 | `1234` | 输入数值，自动校验范围 |
| Int32 / UInt32 | `50000` | 自动使用 FC16 写两个寄存器 |
| Float32 | `3.14` | 按 IEEE754 编码写入 |
| Hex | `1A2B` 或 `0x1A2B` | 十六进制解析 |

### 6.4 输入校验

- 写入弹窗内输入超范围值（如 UInt16 输入 `70000`）→ 弹窗直接提示取值范围
- 设了公式的行写入时，弹窗提示"原始值，不经公式转换"（写入的是 Modbus 原始值，不做反向换算）

## 七、公式转换与仪表盘

### 7.1 行设置弹窗

点击公式列的 **「设置」** 按钮，弹出行设置窗口：

| 字段 | 说明 | 示例 |
|------|------|------|
| **名称** | 数据点名称，填写后将加入仪表盘 | `电池电量`、`温度` |
| **单位** | 工程单位 | `%`、`A`、`V`、`℃` |
| **系数 k** | 线性公式斜率 | `0.1` |
| **偏移 b** | 线性公式截距 | `-40` |
| **小数位** | 显示小数位数，留空则自动 | `1` |

### 7.2 线性公式

```
显示值 = k × 解析值 + b
```

**应用场景示例：**

| 场景 | 解析值范围 | 公式 | 显示值 |
|------|-----------|------|--------|
| 电池电量 0~1000 → 0~100% | 500 | k=0.1, b=0 | 50% |
| 温度传感器 0~4095 → -40~125℃ | 2048 | k=0.04, b=-40 | 41.9℃ |
| 电流变送器 0~2000 → 0~200A | 1500 | k=0.1, b=0, 小数位 1 | 150.0A |

### 7.3 仪表盘视图

- 点击 **「仪表盘」** 按钮切换到大数字卡片视图
- **只有设置了名称的行**才会在仪表盘显示
- 仪表盘卡片显示：**名称 + 大号数值 + 单位**
- 数值变化时卡片黄色闪烁提示
- 单位为 **%** 且值在 **0~100** 之间的卡片显示**进度条**
- 点击 **「表格」** 切换回表格视图
- 无命名行时仪表盘显示引导提示

### 7.4 行设置持久化

行设置（类型、字序、名称、公式）随监控配置自动保存。重启应用、对同一区域/地址/数量开始监控时自动恢复。

## 八、地址模式切换

右上角 **「PLC 地址(4xxxx)」** 勾选框用于一键切换地址显示方式：

| 模式 | 说明 | 示例 |
|------|------|------|
| **PLC 地址**（勾选） | 按 PLC 工程师习惯显示：线圈 0xxxx、离散输入 1xxxx、输入寄存器 3xxxx、保持寄存器 4xxxx | `40001` |
| **协议地址**（不勾选） | 从 0 起始的原始协议地址 | `0` |

切换仅影响显示，不影响实际通信地址。

## 九、断线重连机制

ModbusMate 内置自动重连机制，适用于设备临时断电、网线松动等场景：

```
轮询连续 3 次读失败 → 判定断线 → 状态灯变红
  → "连接中断，自动重连中…"
  → 每 5 秒自动尝试重连
  → 恢复后状态灯变绿，继续监控
```

重连期间你无需任何操作。恢复后日志显示"连接已恢复，继续监控"。

## 十二、设备监控（v0.2 / v0.3）

> 设备模式支持自定义设备类型，按点位模板管理多个设备实例并发采集，按设备分组展示仪表盘。

### 12.1 导航页结构（v0.3）

左侧导航栏五个页面：

| 页面 | 功能 | 说明 |
|------|------|------|
| **📋 设备总览** | 已启动设备的仪表盘 | 按设备分组显示点位大数字卡片，支持启停 |
| **🔧 设备调试** | 单设备调试点位表 | 下拉选择设备，查看各点位原始值/解析值 |
| **🧪 调试工作台** | v0.1 全部功能 | 单设备连接、表格监控、写入、公式、仪表盘 |
| **📜 通信日志** | 全量日志 | 所有设备的通信记录 |
| **⚙️ 类型/实例管理** | 管理配置 | 增删改设备类型和实例 |

### 12.2 首次启动示例数据（v0.3）

首次启动（无 `config.json` 或配置为空）时，应用自动填充示例数据：

- **电池柜**类型（5 个点位：电压/电流/温度/心跳/SOC）
- **PLC 控制器**类型（4 个点位：速度/良品数/温度/运行时长）
- **1号电池柜**实例 → `127.0.0.1:8502`
- **PLC-1**实例 → `127.0.0.1:8503`

配合内置模拟器直接使用：

```bash
# 终端 A — 启动电池柜模拟器
npm run sim:bat

# 终端 B — 启动 PLC 模拟器
npm run sim:plc

# 终端 C — 启动应用
npm start
```

应用到 **「⚙️ 类型/实例管理」** 页可看到预置的类型和实例，点 **「启动」** 后切到 **「📋 设备总览」** 查看仪表盘。

### 12.3 设备类型（模板）

设备类型定义了一组点位模板，一个类型可对应多个设备实例。

**创建类型**：左侧导航 → **「⚙️ 类型/实例管理」** → 点 **「＋ 新建类型」** → 填写类型名称 → 添加点位。

**点位字段**：

| 字段 | 说明 | 取值 |
|------|------|------|
| **名称** | 数据点中文名，必填 | `电池电量` |
| **区域** | Modbus 功能区 | 保持寄存器 / 输入寄存器 / 线圈 / 离散输入 |
| **地址** | 协议地址（0 起） | 0~65535 |
| **类型** | 数据类型 | UInt16 / Int16 / Int32 / UInt32 / Float32 / Hex |
| **字序** | 32 位类型字序 | AB / BA |
| **k / b** | 线性公式系数 | 显示值 = k × 解析值 + b |
| **小数位** | 显示精度，留空自动 | 0~6 |
| **单位** | 工程单位 | %、℃、A、V 等 |

- 支持增删行列点，保存时校验名称必填、地址范围、k/b 数字
- 保存前用 ReadPlan 预演读取块数，>8 块时提示"点位过于分散"

### 12.4 设备实例

设备实例关联一个类型，绑定实际连接参数。

**创建实例**：在 **「⚙️ 类型/实例管理」** 页点 **「＋ 添加设备」** → 填写实例名、选择类型、IP/端口/从站 ID/周期。

- 实例列表显示状态灯 + 实例名 + 类型名 + IP
  - 🟢 已连接 / ⚪ 未启动 / 🔴 离线 / 🟡 重连中
- 点击 **「启动」** → 自动按类型点位算读取规划 → 建立 Modbus-TCP 连接 → 开始周期采集
- 点击 **「停止」** → 断开连接并停止采集
- 点击 **「删除」** → 需先停止再删除

### 12.5 按设备分组仪表盘

每个已启动的实例在 **「📋 设备总览」** 页显示一个设备分组：

- **组头**：状态灯 + 设备名 + IP + 在线/离线 Pill
- **组体**：该类型全部点位的大数字卡片（tile 样式）
- 卡片：名称 + 大号数值 + 单位，单位为 % 且值在 0~100 时显示进度条
- 数值变化时卡片黄色闪烁提示
- **设备离线**时组头状态灯变红（🔴），组体半透明，左侧红色竖线
- 恢复后自动回绿，数值继续更新

### 12.6 多实例并发

多个设备实例独立连接、独立轮询，互不影响。一个设备离线不影响其他设备的采集。

### 12.7 设备调试页

左侧导航 → **「🔧 设备调试」** → 下拉选择设备 → 显示点位明细表：

- 点位名 / 区域 / 协议地址 / 类型 / 原始值(hex) / 解析值
- 底部迷你日志条显示该设备轮询日志

## 十三、内置模拟器

项目附带本地 Modbus-TCP 从站模拟器，用于开发和验证（无需真实设备）：

```bash
npm run sim:bat      # 电池柜（8502，电压/电流/温度/心跳/SOC，动态数据）
npm run sim:plc      # PLC 控制器（8503，速度/良品数/温度）
npm run sim:temp     # 温控器（8504，温度/湿度，趋近目标值）
npm run sim:all      # 同时启动三个
npm run sim          # 旧版基础模拟器（8502，静态数据）
```

### 电池柜模拟器（推荐）

内置三种 profile，数据每秒动态更新：

| Profile | 内容 | 动态行为 |
|---------|------|---------|
| **电池柜** | 电压(×0.1V)、电流(×0.01A)、温度(F32)、心跳、SOC(%) | 电压放电曲线波动、温度缓慢变化、心跳+1 |
| **PLC 控制器** | 速度(rpm)、良品数(U32)、温度(F32 BA)、运行时长 | 速度波动、良品数递增、温度变化 |
| **温控器** | 温度(F32)、湿度(F32)、目标温度 | 温度趋近目标值、加热/制冷线圈自动切换 |

### 旧版基础模拟器

兼容 v0.1 的基础模拟器，预置数据固定不变：

| 地址 | 值 | 说明 |
|------|-----|------|
| 40001 | 250 | 固定值 |
| 40003–40004 | 25.6（Float32 AB 字序） | 32 位浮点示例 |
| 40011 | 每秒 +1（循环） | 模拟传感器变化值，用于观察高亮 |

## 十五、常见问题

### Q1: 连接不上设备？

- 确认设备 IP 与本机网络可达（`ping 设备IP`）
- 确认设备已开启 Modbus-TCP 服务（默认端口 502）
- 确认从站 ID 与设备设置一致
- 检查防火墙是否放行端口 502
- 查看日志栏的具体错误原因

### Q2: 读取的数值明显不对？

- 检查数据类型是否正确（例如整数读成浮点数）
- 检查字序（AB/BA）是否匹配设备
- 检查起始地址是否正确

### Q3: 解析值显示"缺下一寄存器"？

当前行设为 32 位类型，但寄存器数组在行索引+1 处没有数据。减少起始地址或扩大读取数量。

### Q4: 写入失败？

- 只读区域（输入寄存器、离散输入）不支持写入
- 确认写入值在类型范围内
- 查看日志栏的错误提示

### Q5: 重启后配置没恢复？

配置存储在系统 `userData` 目录下的 `config.json`。如果该文件损坏，删除后重启应用即可生成新配置。
