# ModbusMate v0.3 UI 美工设计方案 — IoT 控制台风格

**日期：** 2026-07-03
**视觉稿：** `docs/design/mockups/modbusmate-console.html`（浏览器直接打开）
**适用范围：** 设备总览、设备调试页为主，调试工作台（v0.1 表格）换肤复用同一套 token

---

## 一、设计定位

参照主流 IoT 平台控制台（ThingsBoard / 阿里云 IoT / 涂鸦）的**深色工业监控风**：

- 深色底：长时间盯屏不刺眼，数值与状态灯对比强烈，符合工控看板使用习惯
- 卡片化信息组织：设备为组、点位为卡
- 状态优先：任何界面第一眼能回答"哪些设备在线、哪些出问题了"
- 克制的色彩：数据色和状态色各司其职，图形永远配文字（色盲/打印安全）

## 二、信息架构（v0.3 导航重构）

```
左侧全局导航（窄栏 200px，可折叠为 56px 图标栏）
├── 📟 设备总览      ← 默认首页：按设备分组的实时仪表盘
├── 🔧 设备调试      ← 选中某实例：点位表格 + 写入操作
├── 🧪 调试工作台    ← v0.1 通用寄存器调试（原"调试"模式整体迁入）
├── 📜 通信日志      ← 全局日志（原底部日志栏升级为独立页，各页保留迷你日志条）
└── ⚙️ 类型/实例管理  ← 设备类型点位表编辑、实例增删
```

## 三、页面设计

### 3.1 设备总览（首页）

```
┌────────┬──────────────────────────────────────────────────┐
│        │ 顶栏：ModbusMate │ ●在线 3 ●离线 1 │ [+ 添加设备]   │
│ 全局    ├──────────────────────────────────────────────────┤
│ 导航    │ ┌─ 1号电池柜 ─ 192.168.1.10 ─ ●在线 ─ 1s ─ [⏸] ─┐ │
│        │ │ [电池电量85%▓] [电流14.2A] [电压220V] [温度41.9℃]│ │
│        │ └────────────────────────────────────────────────┘ │
│        │ ┌─ 2号电池柜 ─ ●离线·自动重连中（整组半透明+红边）─┐ │
│        │ │ [── %] [── A] [── V]（保留末次值置灰可选）      │ │
│        │ └────────────────────────────────────────────────┘ │
└────────┴──────────────────────────────────────────────────┘
```

- **设备组卡**：组头 = 设备名 + IP:端口 + 状态 pill + 采集周期 + 启停按钮；组体 = 点位 stat tile 网格（`repeat(auto-fill, minmax(180px, 1fr))`）
- **点位 stat tile**：名称（次级墨色 13px）→ 大数值（28px semibold）+ 单位（14px 弱化）→ 可选进度条（单位 % 且 0~100）；数值变化时卡片描边闪一次强调色（600ms）
- **离线表现**：组头 pill 变 critical（●图标+「离线·重连中」文字），组体 45% 透明度；恢复自动回绿。状态永远是 图标+颜色+文字 三重表达
- 空态：居中引导插画位 +「添加第一台设备」按钮

### 3.2 设备调试页

```
┌ 设备头：1号电池柜 ● 在线 │ 192.168.1.10:502 从站1 │ 周期1s │ [⏸停止] ┐
├ 点位表：名称│区域│地址│类型│原始值│解析值(公式后)│[写入]            ┤
├ 迷你日志条（最近5条，点击跳全局日志页）                              ┤
```

- 表格数值列用 `tabular-nums` 等宽数字对齐；原始值恒 `0xXXXX` 十六进制、Mono 字体
- 写入弹窗沿用现有交互，视觉换新 token；只读点位不渲染写入按钮
- 行变化高亮：整行底色闪强调色 8% 透明度

### 3.3 调试工作台 / 类型管理 / 通信日志

沿用现有功能与布局，仅换肤：同一套色彩/字体/间距/组件 token。类型管理的点位编辑表格 = 设备调试表格同款样式的可编辑态。

## 四、色彩系统（已跑 validate_palette.js 验证）

> 数据色（深色面板，surface `#1a1a19`）：6 色全部通过明度带/色度/对比度检查；CVD 相邻 ΔE 10.3 属"须配文字直标"区间——本设计所有数据图形均带文字标签，满足缓解规则。状态色为保留色，永不用作数据系列。

### 基础面板（深色为默认模式）

| Token | 值 | 用途 |
|---|---|---|
| `--page` | `#0d0d0d` | 页面底 |
| `--surface` | `#1a1a19` | 卡片/表格面板 |
| `--surface-2` | `#232322` | 悬浮层/组头/输入框 |
| `--border` | `rgba(255,255,255,.10)` | 发丝描边 |
| `--ink-1` | `#ffffff` | 主文字/大数值 |
| `--ink-2` | `#c3c2b7` | 次级文字/标签 |
| `--ink-3` | `#898781` | 弱化文字/轴/占位 |

### 强调与数据色

| Token | 值 | 用途 |
|---|---|---|
| `--accent` | `#3987e5` | 主按钮、选中态、进度条、变化闪烁 |
| `--series-2..6` | `#199e70` `#c98500` `#008300` `#9085e9` `#e66767` | 未来多序列图表备用（固定顺序，不循环） |

### 状态色（固定保留，必须 图标+文字 伴随）

| Token | 值 | 用途 |
|---|---|---|
| `--status-good` | `#0ca30c` | 在线/写入成功 |
| `--status-warning` | `#fab219` | 重连中/降级 |
| `--status-critical` | `#d03b3b` | 离线/错误 |

### 浅色模式（预留，本期不实现）

surface `#fcfcfb` / page `#f9f9f7` / ink `#0b0b0b`/`#52514e`/`#898781` / accent `#2a78d6`；状态色同值（浅底上 warning 对比不足由图标+文字缓解）。

## 五、字体与排版

| 场景 | 规格 |
|---|---|
| 全局字体 | `system-ui, -apple-system, "Segoe UI", "Microsoft YaHei", sans-serif`（不引外部字体） |
| 点位大数值 | 28px / 600 / `--ink-1`，比例数字（默认） |
| 表格数值列 | 13px + `font-variant-numeric: tabular-nums` |
| 原始值/Hex/日志 | `ui-monospace, Menlo, Consolas, monospace` 12px |
| 组标题/设备名 | 15px / 600；页面标题 18px / 600 |
| 正文/标签 | 13px / 400；辅助说明 12px `--ink-3` |

## 六、间距 / 圆角 / 层次

- 间距基数 4px：卡内 16、卡间 14、组间 20、页边 24
- 圆角：卡片 10px、按钮/输入 6px、pill 999px
- 深色模式不用投影表现层级，用 **面板色阶 + 发丝描边**（`--surface` → `--surface-2` → border）
- 按钮层级：主按钮实心 `--accent` / 次按钮描边透明底 / 危险操作描边 `--status-critical`

## 七、状态与动效（克制原则：只有数据变化和状态迁移才动）

| 场景 | 表现 |
|---|---|
| 数值变化 | tile 描边闪 `--accent` 600ms（`box-shadow` 过渡），不整卡变底色 |
| 进度条 | 宽度 `transition .3s` |
| 设备离线 | 组体 opacity .45 + 组卡左侧 3px `--status-critical` 竖条 |
| 重连中 | 状态 pill 内圆点 1.2s 呼吸闪烁（`--status-warning`） |
| 空态 | `--ink-3` 引导文案 + 主按钮 |
| 加载 | 骨架条（`--surface-2` 微光扫过），不用转圈遮罩 |

## 八、无障碍与工程约束

1. 状态信息三重编码：颜色 + 图标（●/▲/■ 或 emoji）+ 中文文字，色盲可用
2. 正文对比 ≥ 4.5:1（`--ink-2` on `--surface` 达标），大数值用 `--ink-1`
3. 文字永远用墨色 token，不用数据色/状态色写正文数值
4. CSP `default-src 'self'` 维持——不引外部字体/图标库，图标用系统 emoji 或内联 SVG
5. 全部颜色以 CSS 自定义属性交付（`:root` 一处定义），换肤/浅色模式零改动组件
