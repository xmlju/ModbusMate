# Web 调试入口与全页面 RTU 接入 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 通过 `npm run web` 在本机浏览器启动 ModbusMate，并让调试工作台、设备总览、设备调试、通信日志、类型/实例管理全部能够配置、展示和使用 TCP 或 RTU 设备。

**Architecture:** 新增仅监听 `127.0.0.1` 的 Node HTTP 服务，复用现有 `ModbusService`、`Poller`、`DeviceManager` 和配置校验；浏览器通过带随机会话令牌的严格白名单 RPC 与 SSE 事件流调用本地服务。渲染层保留现有 `window.api` 接口，Electron 继续使用 preload，浏览器由 `web-api.js` 提供同形桥接，避免维护两套页面逻辑。

**Tech Stack:** Node.js CommonJS 内置 `http`/`events`/`crypto`、原生 EventSource/fetch、Electron preload、Vitest 3、现有 HTML/CSS/JavaScript。

- [ ] **Step 3: 实现独立配置存储**

```js
function createConfigStore(filePath, io = fs) {
  return {
    load() {
      if (!io.existsSync(filePath)) return {}
      try { return JSON.parse(io.readFileSync(filePath, 'utf8')) }
      catch (cause) { throw new Error(`配置文件解析失败：${filePath}`, { cause }) }
    },
    save(config) {
      io.mkdirSync(path.dirname(filePath), { recursive: true })
      const temp = `${filePath}.tmp`
      io.writeFileSync(temp, JSON.stringify(config, null, 2), 'utf8')
      io.renameSync(temp, filePath)
      return { ok: true }
    },
  }
}
```

- [ ] **Step 4: 测试 Web 运行时事件和关闭顺序**

- [ ] **Step 3: 实现静态文件、RPC 和 SSE**

服务必须：

```js
const WEB_RPC_CHANNELS = new Set([
  'serial:list', 'modbus:connect', 'modbus:disconnect', 'modbus:startPoll',
  'modbus:stopPoll', 'modbus:write', 'config:load', 'config:save',
  'device:start', 'device:stop', 'device:write',
])
```

- 仅监听 `127.0.0.1`，默认端口 `8765`，测试允许端口 `0`。
- 每次启动生成至少 32 字节随机 token；RPC 要求 `X-ModbusMate-Token` 和精确同源 `Origin`。
- SSE 因 EventSource 不能自定义 header，使用查询参数 token，并设置 `Cache-Control: no-store`。
- JSON 请求体最大 1 MiB；未知通道、路径穿越、非 JSON 和超限请求返回中文结构化错误。
- 静态资源只来自 `renderer/`，CSP 禁止外部脚本、对象和 frame。

## 最终验收

- `npm run web` 在浏览器打开本地页面，不依赖 Electron 安装包。
- 工作台和设备实例可选择 TCP/RTU，旧 TCP 配置自动兼容。
- 所有五个页面正确显示 TCP 地址或 RTU 串口摘要。
- RTU 能执行连接、读、写、轮询、停止与重连；一个从站独占一个串口。
- macOS 与 Windows 串口可刷新选择，错误提示能区分不存在、占用、权限和从站无响应。
- Web RPC 只监听回环地址，使用随机 token、同源校验、白名单和请求体上限。
- 全量自动测试、Web 冒烟、macOS/Windows 构建冒烟全部通过。
- 最后使用真实 USB-RS485 设备按点表进行人工读写和拔插验收。
