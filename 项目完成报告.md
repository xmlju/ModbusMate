## 一、项目总览

ModbusMate 是一款 Electron 桌面端 Modbus-TCP 调试工具，面向现场工程师和调试人员，支持四种功能区轮询监控、六种数据类型编解码、单点写入、线性公式换算、仪表盘大数字卡片视图、断线自动重连，以及 v0.2 新增的设备模型与多设备并发采集。

## 二、架构分层

```
渲染层（renderer/）
├── index.html      → 界面结构（调试/设备双模式 + 5 个弹窗）
├── style.css       → 样式（深色主题/状态灯/变化闪烁/设备模式布局）
├── app.js          → 调试模式交互 + 模式切换钩子
├── device.js       → 设备模式 UI（类型/实例管理 + 分组仪表盘，v0.2 新增）
├── codec.js        → 纯函数编解码 + 地址换算 + 线性公式（无依赖）
└── read-plan.js    → 点位批量读取规划（纯函数，v0.2 新增）

桥接层
└── preload.js      → IPC 白名单（v0.1 调试 API + v0.2 设备 API）

主进程（main/）
├── index.js           → 窗口管理 + IPC 注册 + 配置持久化 + 崩溃日志
├── modbus-service.js  → Modbus-TCP 连接/读/写（异常码中文翻译）
├── poller.js          → 单设备轮询调度 + 写入队列 + 断线重连
├── device-manager.js  → 多设备实例并发采集（v0.2 新增）
└── activation.js      → 激活验证（当前已注释禁用，始终跳过）

测试（test/）
├── codec.test.js          → 编解码 31 个用例
├── modbus-service.test.js → 服务层 7 个用例（Mock 模拟 TCP）
├── poller.test.js         → 轮询调度 4 个用例
├── activation.test.js     → 激活 4 个用例（简化版）
├── read-plan.test.js      → 读取规划 6 个用例（v0.2 新增）
├── device-manager.test.js → 设备管理 5 个用例（v0.2 新增）
└── simulator.js           → Modbus-TCP 模拟从站

服务端（scf/ + workers/）
├── scf/index.js        → 腾讯云 SCF 主节点激活服务
├── workers/index.js    → Cloudflare Worker 备用节点激活服务
└── scripts/generate-codes.js → 激活码生成脚本
```

## 四、质量保障

- **TDD 全覆盖**：每项功能先写测试确认失败，再实现代码
- **Mock 隔离**：`modbus-service.test.js` 使用 `sinon` Mock 库；`device-manager.test.js` 使用 stub service 隔离网络
- **边界防护**：空输入/Infinity/非法寄存器/超范围写入/未知区域类型均有校验
- **CSP 安全**：`default-src 'self'` 限制渲染层资源加载
- **崩溃日志**：主进程捕获 `uncaughtException` + `unhandledRejection` 写入文件
- **中文友好**：所有错误提示面向现场工程师，不暴露技术细节
