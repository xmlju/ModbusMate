## 1. 数据模型契约（#5 的基础，先读这一节）

点位对象当前字段：`{ name, area, addr, type, wordOrder, k, b, decimals, unit }`。

**新增一个字段 `visible`**：
- 类型：boolean，**缺省视为 `true`**（老数据没有该字段时按显示处理，保证向后兼容）。
- 判断显示用 `p.visible !== false`（而不是 `p.visible === true`）。

**排序 = `type.points` 数组顺序**：数组里的先后即显示先后，不引入单独的 order 字段。上移/下移就是交换数组相邻元素。

**影响范围**：`visible` 和排序**只影响“显示”，不影响“采集”**。轮询仍按 ReadPlan 读取全部点位所在的寄存器块（隐藏点位所在寄存器可能和可见点位在同一块里，照读不误），只是不在界面上渲染卡片/行。**不要改 ReadPlan、不要改 main/**。

**需要保留 `visible` 的函数**（都在 renderer/）：
- `codec.js` 的 `validatePoints`：导入点表时保留/规范化 `visible`（缺省 true）。导出点表时带上 `visible`。
- `device.js` 的 `collectPointsFromTable`：从编辑表格收集时读取“是否显示”勾选写入 `visible`。

## 2. 需求 #1：类型编辑表格改造（`device.js` renderTypePoints/addPointRow + `index.html` 表头 + `style.css`）

类型编辑弹窗（`#typeEditorModal`）的点位表格改造：

### 1.1 点位名称筛选
- 表格上方加一个筛选输入框（placeholder「按名称筛选点位」）。
- 输入时**实时过滤**表格行（不区分大小写，匹配 name 包含即显示）。
- 筛选只影响显示，不影响数据；清空恢复全部。
- 注意：筛选状态下做上移/下移/删除，仍要正确对应到 `type.points` 里的真实索引（不要用可见行的下标去改数组）。

### 1.2 区域列标注功能码
区域下拉/显示旁标注对应 Modbus 功能码：
| 区域 | 读 | 写 |
|---|---|---|
| 保持寄存器 holding | 03 | 06 / 16 |
| 输入寄存器 input | 04 | 只读 |
| 线圈 coil | 01 | 05 |
| 离散输入 discrete | 02 | 只读 |

下拉选项文案改为如「保持寄存器 (03/06)」「输入寄存器 (04·只读)」「线圈 (01/05)」「离散输入 (02·只读)」。

### 1.3 地址进制切换
- 表格上方加勾选框「16 进制地址」。
- 勾选后地址列显示为 `0x` 十六进制（如 `0x2710`），**编辑时输入也按 16 进制解析**；不勾选为 10 进制（现状）。
- 这是**独立于**工作台已有「PLC 地址(4xxxx)」的显示开关，两者不冲突，本项只作用于类型编辑表格。
- 保存到数据里始终是 10 进制协议地址（进制只是显示/输入形态）。

### 1.4 k / b 列
- 两列**变窄**（当前偏宽）。
- 表头 k、b 各加一个「?」说明按钮，点击弹出说明气泡/提示：`显示值 = k × 解析值 + b`（k 系数、b 偏移）。可用 title 或小弹层实现。

### 1.5 单位列变窄。

### 1.6 操作列增加「是否显示」
- 操作列（当前是空表头的最后一列）加一个勾选框，绑定该点位 `visible`。
- 默认勾选（显示）。取消勾选 → 该点位在总览/设备调试/工作台仪表盘不显示（见 #5）。

### 1.7 上移 / 下移
- 操作列加「↑」「↓」按钮，交换该点位与相邻点位在 `type.points` 里的位置，重渲染表格。
- 第一行的↑、最后一行的↓ 置灰/禁用。

> 表头顺序建议：名称 | 区域 | 地址 | 类型 | 字序 | k | b | 小数位 | 单位 | 显示 | 操作(↑↓删)

## 3. 需求 #5：所有点位显示页面遵循 visible + 排序

以下渲染函数（都在 `renderer/`）在遍历点位时，**过滤 `p.visible !== false`，并按 `type.points` 数组顺序**：
- `device.js` `renderOverviewPage`（设备总览分组卡片）
- `device.js` `renderDebugForDevice`（设备调试点位表）
- `app.js` 工作台仪表盘 `renderDashboard`（如该视图基于点位/行）

隐藏的点位不渲染卡片/行；顺序与类型编辑表格里一致。

## 5. 需求 #3：调试工作台

### 5.1 波特率固定推荐下拉
- 工作台连接栏 `#baudRate`（和实例弹窗 `#instBaudRate`）波特率改为固定下拉选项：**2400 / 4800 / 9600 / 19200 / 38400 / 57600 / 115200**（默认 9600）。
- 实例弹窗的 `instBaudRate` 当前是 number 输入，改成 select 与工作台一致。

### 5.2 输入项对齐
- 连接栏各输入项（通信方式、IP/端口 或 串口参数、从站ID、超时）尽量对齐显示，label 与输入框宽度统一，视觉整齐。用 flex 对齐即可。

### 5.3 「点位与类型定义一致」——已确认含义（2026-07-15 姚确认）

**含义 = 显示逻辑统一**：点位配置表（类型里的点位定义）是**唯一数据源**，它设置的「是否显示」「排序」「点位个数」，在所有显示该设备点位的页面上都必须一致——包括**调试工作台**在内。

即：本条不是独立功能，而是把 §3（#5）的“遵循 visible + 排序 + 个数”范围**明确扩大到调试工作台的点位显示**。

**实现要求**：凡是渲染某设备/类型点位的地方，一律：
- 只显示 `p.visible !== false` 的点位；
- 按 `type.points` 数组顺序显示；
- 点位个数以配置表为准（配置几个显示几个，不多不少）。

工作台若基于自身 `state.rows` 显示，也要接入同一套 visible/排序/个数规则，做到和总览、设备调试完全一致。
