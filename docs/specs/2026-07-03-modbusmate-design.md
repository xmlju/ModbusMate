**日期：** 2026-07-03

**状态：** 已确认

## 一、产品定位

ModbusMate 是一款**通用 Modbus-TCP 调试工具**（桌面应用，Windows + Mac），面向现场工程师和调试人员分发：

1. **轮询监控**：连接任意 Modbus-TCP 设备，按周期轮询寄存器区域，实时表格展示
2. **指令下发**：单点写入线圈（FC05）和保持寄存器（FC06），表格内直接操作
3. **数据解析**：Int16 / UInt16 / Int32 / UInt32 / Float32（AB/BA 字序）/ Hex
4. **激活授权**：激活码 + 设备绑定 + 服务端验证（本项目独立部署，与其他产品零耦合）

**独立项目**：独立目录、独立 git 仓库、独立激活服务，与 MicroDramaHelper / moviestudio 无任何依赖。

## 三、整体架构

```
┌─────────────────────────────────────────────┐
│ 渲染进程（界面）                              │
│  连接管理 │ 监控表格 │ 写入操作 │ 激活界面     │
└──────────────────┬──────────────────────────┘
                   │ IPC（contextBridge，预加载白名单）
┌──────────────────┴──────────────────────────┐
│ 主进程（Node.js）                            │
│  Modbus 服务   │  轮询调度器   │  激活服务    │
│ (modbus-serial)│ (定时读+写队列)│ (Token 验证) │
└─────────────────────────────────────────────┘
          │ TCP :502                │ HTTPS
     Modbus 设备              激活验证服务（SCF/Worker）
```

**安全边界**：`contextIsolation: true`，渲染进程无 Node 权限，所有 Modbus / 文件 / 网络操作经预加载脚本白名单 IPC 走主进程。

**职责划分**：

- **Modbus 服务**（`main/modbus-service.js`）：连接、读（FC01–FC04）、写（FC05/FC06）、断线重连；单连接单设备（本期不做多设备并发）
- **轮询调度器**（`main/poller.js`）：按周期批量读区域，推送数据给渲染层；维护写入优先队列
- **激活服务**（`main/activation.js`）：启动验 Token，未激活弹激活流程
- **数据持久化**：连接配置、监控区域配置、deviceId 存 `userData` 目录 JSON 文件，重启自动恢复

## 四、界面设计

主窗口单页布局（约 900×640，可缩放），自上而下四个区域：

```
┌──────────────────────────────────────────────────────────┐
│ 连接栏： IP │ 端口(502) │ 从站ID(1) │ [连接] │ ●状态灯     │
├──────────────────────────────────────────────────────────┤
│ 监控配置： 区域类型▾ │ 起始地址 │ 数量 │ 周期▾ │ [▶开始]   │
├──────────────────────────────────────────────────────────┤
│ 监控表格： 地址 │ 数据类型▾ │ 原始值 │ 解析值 │ [写入]      │
├──────────────────────────────────────────────────────────┤
│ 日志栏（可折叠）：通信记录 + 错误信息，最近 200 条           │
└──────────────────────────────────────────────────────────┘
```

### 交互细节

- **连接**：状态灯 灰=未连接 / 绿=已连接 / 红=错误；连接参数自动记忆
- **监控**：区域类型四选一（线圈 FC01 / 离散输入 FC02 / 保持寄存器 FC03 / 输入寄存器 FC04）；轮询周期默认 1000ms，可选 100ms–10s；数量上限 120（协议单次上限 125 寄存器，留余量）
- **变化高亮**：数值变化的单元格黄色闪烁提示
- **数据类型**：每行独立设置；32 位类型自动占两个寄存器（下一行合并显示）
- **写入**：点行内「写入」弹输入框 → 按该行类型编码 → FC06（寄存器）/ FC05（线圈）下发 → 立即回读确认；只读区域（FC02/FC04）不显示写入按钮
- **地址模式**：PLC 习惯地址（40001 起）/ 协议地址（0 起）全局切换
- **激活**：未激活时全窗口遮罩 + 激活码输入（格式 XXXX-XXXX，自动加横线）

## 五、Modbus 通信与数据流

### 连接与容错

- `connectTCP()`，连接超时 5s，请求超时 2s（设置可调）
- **断线重连**：轮询连续失败 3 次判定断线 → 状态灯变红 → 每 5s 自动重试 → 恢复后自动继续监控

### 轮询数据流

```
定时器到期 → 按区域功能码整块批量读
  → 成功：原始 Uint16Array → IPC 推送渲染层 → 按行解析 → 刷新表格
  → 失败：日志栏记录，失败计数 +1（3 次触发重连）
```

- 解析放渲染层：主进程只传原始值，数据类型/字序是纯显示层概念，切换类型不重新请求

### 写入队列

- 写请求优先级高于轮询：当前读完成后立即执行写，写后立刻补读回——「写入 → 表格见新值」一个周期内闭环
- 输入校验（类型范围、格式）在界面层完成，主进程只编码下发

### 数据类型编解码（`renderer/codec.js`，纯函数）

| 类型 | 寄存器数 | 说明 |
|---|---|---|
| Int16 / UInt16 | 1 | 原生 |
| Int32 / UInt32 / Float32 | 2 | 字序 AB（高字在前）/ BA（低字在前）可选 |
| Hex | 1 | 十六进制显示，写入也按 hex 解析 |

### IPC 通道

- 渲染 → 主：`modbus:connect / disconnect / startPoll / stopPoll / write`，`activation:verify`
- 主 → 渲染：`modbus:data / status / log`（事件推送）

## 六、激活授权

本项目**独立部署**一套激活验证服务（参考已验证的 HMAC 方案实现，代码归属本项目）：

### 客户端流程

1. 启动读本地缓存（`userData/activation.json`）：`code / token / expires`
2. 本地 HMAC 验签 Token（无需联网），30 天有效期 + 7 天断网宽限
3. 无效/过期 → 显示激活遮罩 → 用户输码 → 服务端验证 → 缓存新 Token
4. `deviceId`：`crypto.randomUUID()` 生成后持久化到 `userData`

### 服务端（`scf/` 主节点 + `workers/` 备用节点，双节点依次重试）

- `POST /verify`：验激活码 HMAC → 查吊销名单 → 设备绑定检查（一码一设备）→ 签发 30 天 Token
- `POST /revoke`：管理员吊销激活码
- 环境变量：`SECRET`（激活码密钥）、`SIGN_KEY`（Token 签名密钥）、`ADMIN_KEY`；本产品**独立密钥**，与任何其他产品的激活码互不通用

### 激活码

- 格式 `XXXX-XXXX`：base32 序列号 + HMAC 校验码，字符集去除 0/O/I/l
- `scripts/generate-codes.js` 离线批量生成，`codes.txt` 不入库

## 七、错误处理

- **面向现场工程师的详细错误提示**（中文，带原因和建议）：
  - 连接失败：「目标主机 192.168.1.10:502 无响应，请检查设备电源和网线」
  - Modbus 异常码全部翻译：如异常码 02 →「非法数据地址，该设备可能不存在地址 40100」
- 通信错误进日志栏（含时间戳），崩溃级错误写本地日志文件（`userData/logs/`）便于远程排查
- 激活失败区分提示：码无效 / 已被其他设备使用 / 网络不通
